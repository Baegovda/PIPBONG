# PIPBONG 업데이트 로그

버전별 변경 사항을 한글로 정리한 문서입니다. 최신 릴리즈는 [GitHub Releases](https://github.com/Baegovda/PIPBONG/releases/latest)에서 받을 수 있습니다.

개발용 상세 기록(영문)은 [`AGENTS.md`](../AGENTS.md) §11을 참고하세요.  
패치 버전은 **10개 단위**로 묶어 표기합니다(예: `v0.8.60–0.8.69`).

## v0.8.300–0.8.313 (2026/07/24)

**수정**

- v0.8.312에서도 남던 **홀드 동시 입력** GUI 멈춤(~300 ms)을 추가로 줄였습니다. 버스트 동안 포그라운드·캡처 동기화 중복을 막고, 종료 처리도 한 번에 하나씩 나눠 실행합니다.

- v0.8.311에서도 Q/W/E/R **홀드 동시 입력** 시 GUI가 1초 이상 멈추던 문제를 추가 수정했습니다. 홀드 시작을 한 프레임에 하나씩 처리하고, 워크플로 복제는 GUI가 아닌 워커에서 하도록 바꿨습니다.

- Q/W/E/R 등 **홀드** 단축키를 **동시에 누르거나 뗄 때** GUI가 오래 멈추던 문제를 추가로 줄였습니다. 여러 홀드가 한꺼번에 시작·종료될 때 포그라운드 준비·화면 갱신·목록 정책 갱신을 한 번만 처리합니다.

- Q/W/E/R 등 **홀드** 기능을 여러 개 동시에 뗄 때 GUI가 수 초 멈추던 문제를 줄였습니다. 여러 세션이 있을 때 실행 UI 갱신을 묶고, 세션 종료 알림·워크플로우 저장을 한꺼번에 처리합니다.

- 기능 목록 **사용** ON/OFF 시 Ctrl+Z용으로 **프로필 폴더 전체**를 복사하던 동작을 없애고, 현재 프로필의 **project.json**만 백업하도록 바꿔 토글이 훨씬 빨라졌습니다.

- LOL **런처↔클라이언트** 창을 빠르게 오갈 때 프로필이 **왔다 갔다(핑퐁)** 하며 멈추던 문제를 줄였습니다. 전환이 필요하면 트리거 **감시를 즉시 중지**한 뒤, 포그라운드가 잠깐 안정된 뒤 **한 번만** 프로필을 바꿉니다.
- 프로필 전환 중 목록 **드래그**와 겹치며 GUI가 오래 멈추던 경우를 줄였습니다. 전환 중에는 목록 재정렬을 막고, 드래그 중에는 자동 전환을 미룹니다.
- **홀드·무한 반복** 빠른 루프 중 `startTimer` 경고가 쌓이던 문제를 줄였습니다(워커 스레드 UI 갱신을 GUI 스레드로 넘김).

**추가**

- (개발용) **홀드 동시 입력 진단**을 **프로그램 설정 → 진단**에서 켤 수 있습니다. Q/W/E/R 등 홀드를 동시에 누름/뗌 때 단계별 ms를 `multi-hold/latest.md`에 기록합니다. `scripts/analyze-multi-hold.ps1`로 요약할 수 있습니다.

- (개발용) **기능 사용 토글 진단**을 **프로그램 설정 → 진단**에서 켤 수 있습니다. 기능 목록 **사용** ON/OFF마다 단계별 ms와 가장 느린 구간을 `feature-toggle/latest.md`에 기록합니다. `scripts/analyze-feature-toggle.ps1`로 요약할 수 있습니다.

- (개발용) **프로필 전환 진단**을 **프로그램 설정 → 진단**에서 켤 수 있습니다. 수동·자동 전환마다 단계별 ms와 가장 느린 구간을 `profile-switch/latest.md`에 기록합니다.
- (개발용) **프로필 전환 진단**: `program/profileSwitchProfiling` 또는 환경 변수 `PIPBONG_PROFILE_SWITCH_PROFILE=1` — 전환 단계별 ms·핑퐁 횟수를 `profile-switch/latest.md`에 기록. `scripts/analyze-profile-switch.ps1`로 확인.

- Q/W/E/R처럼 **홀드** 기능을 **두 개 이상** 동시에 켠 채 오래 쓸 때 PIPBONG이 다시 느려지던 문제를 추가로 줄였습니다. 동시 실행 중에는 반복 루프 실행 로그를 아예 쓰지 않고, 화면 갱신 주기도 세션 수에 따라 늘립니다.
- **커서 스터터 진단**을 켠 상태로 오래 쓸 때 부담을 줄였습니다. 샘플링 간격을 늘리고, 일반 마우스 이동은 숫자만 세며 이벤트 목록에는 넣지 않습니다.

**변경**

- **기능 편집**, **프로필 편집**, **키보드**·**딜레이** 블록 편집기, **프로그램 설정** 등 설정·편집 화면을 **종류별 그룹**으로 묶어 가독성을 높였습니다. 자세한 설명은 각 항목에 마우스를 올리면 툴팁으로 볼 수 있습니다.

- **커서 스터터 진단**을 **앱 스파이크 진단**으로 교체했습니다. PIPBONG 창 버벅임·UI 지연·CPU 스파이크를 기록하며, 커서 위치를 계속 샘플링하지 않습니다. **프로그램 설정 → 진단 → 앱 스파이크 진단**을 켠 뒤 재현하고 앱을 종료하면 `app-spike/latest.md`에 저장됩니다. 평소에는 꺼 두세요.

**제거**

- **커서 스터터 진단**(`cursor-stutter/latest.md`) 체계를 제거했습니다. 앱 지연 분석은 **앱 스파이크 진단**을 사용하세요.

_(사용자 인터페이스 변경 없음 — 에이전트 답변·진단 지침만 추가)_

- 버그·버벅임이 **특정 기능**에서 날 때, 그 **해당 부분만** 재현용 프로파일링을 같은 작업 안에 붙이도록 에이전트 정책을 추가했습니다.

## v0.8.290–0.8.300 (2026/07/24)

**변경**

- **커서 스터터 진단** 로그에 **현재 포커스 창**(창 제목·HWND·실행 파일)이 기록됩니다. 포커스가 바뀔 때마다 `foreground_focus` 이벤트가 남고, 커서 점프·Hold·훅 이벤트에도 같은 시점의 포커스 창이 붙습니다.

- **커서 스터터 진단**이 기본으로 켜지며, 앱 실행 중 **항상** 커서 위치를 샘플링합니다. 8px 이상 점프뿐 아니라 미세 점프·왕복 튐·샘플러 지연도 기록합니다. Hold 기능 **Q/W/E/R** 시작·종료도 로그에 남깁니다. 보고서는 저장소 `cursor-stutter/latest.md`와 AppData 백업 경로에 저장되며, Hold 키를 떼거나 앱 종료 시 갱신됩니다.

**추가**

- **커서 스터터 진단**: QWER 연타 등 마우스 커서가 튀는 현상 원인 분석용 로그. **프로그램 설정 → 진단 → 커서 스터터 진단**을 켠 뒤 재현하고 앱을 종료하면 `cursor-stutter/latest.md`에 저장됩니다.

**제거**

- 기존 **앱 전체 초정밀 프로파일링**(`app-profile/latest.md`) 체계는 제거되었습니다. 커서 튐 진단은 위 **커서 스터터 진단**을 사용하세요.
- 저장소에 남아 있던 구 **`app-profile/`** 로그 파일(`latest.md`, `trace.json`)을 삭제했습니다. 진단은 **`cursor-stutter/latest.md`** 만 사용하세요.

**수정**

- Q/W/E/R처럼 **홀드** 기능을 여러 개 동시에 켜도 PIPBONG 창이 버벅이거나 응답 없음이 뜨던 문제를 줄였습니다. 동시 실행 시 실행 로그·기능 목록 갱신·애니메이션 부하를 묶어서 처리합니다.
- **커서 스터터 진단**이 기본 켜진 상태에서 앱이 심하게 끊기거나 응답 없음이 뜨던 문제를 고쳤습니다. 진단은 이제 기본 **꺼짐**이며, 켰을 때만 샘플러가 동작합니다. Hold 키를 뗄 때마다 로그를 쓰지 않고 **앱 종료 시**에만 저장합니다.
- Alt+Tab으로 다른 창에 갔다가 메인 타겟으로 돌아와도 단축키가 바로 동작합니다. PIPBONG에 한 번 포커스할 필요 없습니다.
- 제목이 비어 있는 전체 화면 게임(리그 오브 레전드 클라이언트 등)에 포커스만 두어도 프로필·캡처·트리거가 바로 동작합니다. PIPBONG을 한 번 클릭할 필요가 없습니다(실행 파일 경로로 창을 연결).
- 대상 창(메인·서브)에 포커스만 두어도 트리거·캡처가 바로 동작합니다. PIPBONG에 한 번 포커스한 뒤 다시 대상 창으로 돌아갈 필요가 없습니다(포그라운드 창의 실행 파일 경로로 메인/서브 캡처를 직접 연결).
- 서브 대상 창에 포커스를 두거나 서브 창만 실행 중일 때 **타겟** 패널이 메인 창 제목과 **미실행**으로 잘못 표시되던 문제를 고쳤습니다. 서브 창이면 **서브 창** 배지와 실행 정보가 표시됩니다.
- 대상 창(메인·서브) 포커스를 더 자주 감지해 **타겟** 패널이 즉시 갱신됩니다. PIPBONG으로 포커스를 옮겨도 방금 연결했던 대상 창 정보가 유지됩니다.

## v0.8.280–0.8.289 (2026/07/23)

**변경**

- 여러 기능을 동시에 실행할 때(예: Q/W/E/R 홀드) 키 입력이 서로 겹쳐도 Windows 입력 대기가 덜 쌓이도록 입력 주입 경로를 정리했습니다.
- 워크플로 실행 프로파일(표준 모드)에서 반복 루프마다 쌓이던 로그를 줄이고, 키 입력 지연은 요약 통계로만 기록합니다.

- 영역 구분선(스플리터) 드래그 히트 영역을 넓혀 패널 크기 조절이 더 쉬워졌습니다.

- 메인 화면 UI를 간소화했습니다. 프로필·기능 목록·워크플로·로그·대상 창·하단 버튼 등에서 겹치던 테두리와 여백을 줄이고, 상태 표시줄·동작 방식 칩·실행 버튼을 더 평면적으로 정리했습니다.
- 프로필·기능 목록 행의 불필요한 카드 테두리를 제거하고 구분선을 옅게 했습니다.
- 제목 표시줄 높이를 낮추고 메뉴·창 버튼 여백을 줄였습니다.
- 영역 구분선(스플리터)은 마우스를 올렸을 때만 표시됩니다(드래그 영역은 그대로 유지).
- 워크플로 블록 목록 **요약** 열: **마우스** 블록에 Ctrl/Alt/Shift 조합키가 켜져 있으면 `Ctrl+Shift+클릭·직전`처럼 함께 표시됩니다.

**수정**

- Alt+Tab으로 다른 창에 갔다가 대상 창으로 바로 돌아와도 기능·단축키가 먹통이 되지 않습니다(PIPBONG을 거쳐 다시 포커스할 필요 없음).

- **사용자 입력 시 완전 정지**가 워크플로 **키보드**·**마우스** 블록이 보낸 입력을 사용자가 누른 것으로 오인해 멈추던 문제를 수정했습니다(프로그램이 입력한 키/버튼은 일정 시간 동안 사용자 입력으로 보지 않음).

- 프로필·기능 목록·대상 창 등 그룹 제목이 아래 목록/내용과 겹치던 문제를 수정했습니다.
- 대상 창 패널에서 창 제목·상태·메타데이터가 좁은 너비에서 겹치던 문제를 수정했습니다(말줄임·툴팁).
- 워크플로 상태줄(프로필 › 기능 › 동작 방식)에서 긴 이름이 겹치지 않도록 말줄임 처리했습니다.
- 기능 목록·워크플로 블록 목록 헤더에서 긴 열 이름(예: 기준/감지)이 이웃 열과 겹치지 않도록 말줄임 처리했습니다.

- 프로그램 창 가장자리를 드래그해 크기를 조절할 수 없던 문제를 수정했습니다.
- 프로필·기능 목록·워크플로·로그/대상 창·라이브러리·블록 목록 등 영역 구분선을 드래그해도 크기가 바뀌지 않던 문제를 수정했습니다.

## v0.8.270–0.8.279 (2026/07/22)

**추가**

- **앱 전체 초정밀 프로파일링**: 로그는 **`app-profile/latest.md`** 한 파일만 갱신됩니다 (`trace.json` 동봉). 예전 `workflow-profile/sessions/` 아카이브는 사용하지 않습니다.

**변경**

- 프로파일링 로그 폴더 **`app-profile/`** 로 변경. AI는 이 경로만 보면 됩니다.
- 프로파일 로그에 **프로필 설정(대상 창·캡처 방식 등)·기능 설정·워크플로 블록 구성·블록별 측정·자동 진단**이 함께 저장됩니다. 재현만 하면 AI가 워크플로를 다시 묻지 않고 분석합니다.
- **프로파일링 상세도**: **표준**(스파이크만), **상세**(매칭/미스 폴 + 물리 입력 + 포그라운드 창), **초정밀**(모든 ImageFind 폴) 선택 가능.
- 물리 키보드·마우스(`user_physical_*`)와 PIPBONG 합성 입력(`synthetic_*`)이 로그에서 구분됩니다.
- ImageFind **폴마다** 캡처/매칭 시간·신뢰도(`imagefind_poll`)가 기록됩니다(상세도에 따라 필터).

**변경**

- 진단 옵션 이름을 **앱 전체 초정밀 프로파일링**으로 바꿨습니다. 기능만이 아니라 앱 전 구간이 한 로그에 누적됩니다.
- 트리거 모드는 감시·동작·쿨다운 단계가 로그에 구분되어 기록됩니다.
- 프로파일 로그 형식이 v5로 확장되었습니다(프로필 스냅샷, 트리거 집계, 포그라운드 타임라인, Chrome Trace JSON).

**수정**

- 프로파일 로그에서 블록 종류(`type=`)가 비어 보이던 문제를 수정했습니다.
- 트리거 **감시** 중 홀드 매크로·다른 기능 실행과 화면 캡처가 겹치며 끊김이 나던 문제: 감시 폴링이 활성 동작 구간에서 잠시 양보하고, 캡처는 한 번에 하나만 수행합니다.
- **창 없이 동작** 프로필에서도 서브/메인 포커스 조건으로 감시가 멈추던 문제를 수정했습니다.
- 감시 엔진이 일시 중단될 때 트리거 감시가 완전히 꺼지던 문제 — 사용자가 멈추지 않았으면 감시를 다시 시작합니다.
