# PIPBONG 업데이트 로그

버전별 변경 사항을 한글로 정리한 문서입니다. 최신 릴리즈는 [GitHub Releases](https://github.com/Baegovda/PIPBONG/releases/latest)에서 받을 수 있습니다.

개발용 상세 기록(영문)은 [`AGENTS.md`](../AGENTS.md) §11을 참고하세요.  
패치 버전은 **10개 단위**로 묶어 표기합니다(예: `v0.8.60–0.8.69`).

---

**변경**

- 워크플로가 실행 중이어도 프로그램 종료 시 확인 창 없이 바로 종료합니다.

- **트리거** 모드 **쿨다운**을 밀리초 대신 **초** 단위(0~600초, 1초 단위)로 설정합니다. 기능 목록 툴팁과 실행 로그도 초로 표시합니다.

## v0.8.180–0.8.189 (2026/07/17)

**추가**

- **트리거** 모드 감시 켜짐/꺼짐 상태가 프로필마다 저장됩니다. 프로그램을 다시 켜거나 프로필을 바꿔도, 중지하기 전까지 켜 둔 트리거 감시는 자동으로 이어집니다.

## v0.8.170–0.8.179 (2026/07/16 – 2026/07/17)

**수정**

- **트리거** 모드에서 감시는 되는데 워크플로가 바로 **트리거 상태 오류**로 실패하던 문제를 고쳤습니다. 감시에서 찾은 매칭 결과가 실행 단계로 넘어가기 전에 지워지지 않도록 했습니다.

- **서브 대상 창**을 쓰는 프로필에서 트리거·기능 실행이 주 대상(게임) 창으로 강제로 바뀌던 문제를 고쳤습니다. 서브 창에 포커스가 있으면 서브를 캡처하고, PIPBONG에서 단축키로 시작할 때도 서브 창이 떠 있으면 서브를 우선합니다.

**변경**

- **트리거** 모드 **쿨다운** 설정(기능 편집): 워크플로 1회 **성공** 후 다음 감시까지 대기 시간입니다. 블록 편집기 **탐지 재시도** 간격(감시 중 매칭 실패 시 재탐색)과는 별개입니다. 실행 실패 시에는 쿨다운 없이 바로 다시 감시합니다.

- **트리거** 모드 실행 표시 색을 따뜻한 골드 대신 차분한 제이드-틸(청록) 톤으로 바꿨습니다. 사용·▶ 버튼, 행 강조, 워크플로 감시 표시, 대상 창 스캔 링이 같은 톤으로 맞춰집니다.

- **트리거** 모드 실행 표시를 일반 실행과 같은 따뜻한 앰버 톤으로 다시 맞추고, 시안색·레이더 아이콘 대신 은은한 행 배경·부드러운 강조로 정리했습니다. 감시 중 **◎**, 쿨다운 **◔** 표시는 유지하며 대상 창 스캔 링도 차분한 골드색으로 바꿨습니다.

- **대상 창** 정보 패널이 로그와의 가로 스플리터 너비에 맞춰 자동으로 줄바꿈·재배열 됩니다. 넓을 때는 한 줄 요약, 좁아지면 항목별 줄바꿈으로 잘리지 않습니다.

- 실행 중인 기능 행 전체에 프리즘 글로우(그라데이션·쉬인·테두리 후광)가 적용되도록 바꿨습니다. 이름 텍스트만 색이 바뀌던 방식에서, ▶ 버튼·사용 토글·방식 열까지 같은 톤으로 맞춥니다.

- 워크플로우 패널 상단: 루프 횟수·직전/평균 소요 시간·성공/실패를 터미널 스타일 칩으로 표시해 한눈에 읽기 쉽게 바꿨습니다.

- 실행 중인 기능 이름 강조: 선명한 무지개 대신 라벤더·하늘·얼음·라일락·로즈 톤이 천천히 흐르는 **프리즘** 스타일(은은한 색수차 글로우)로 바꿨습니다.

- **트리거** 모드 감시 중에는 일반 실행과 다른 시각 피드백: 기능 목록에 청록색 호흡형 **감시** 강조, 쿨다운 중에는 은은한 호박색 표시, 워크플로 블록 목록에 **◎** / **◔** 감시·대기 표시, 대상 창에는 느린 청록색 스캔 링(감시 중 빨간 미스 깜빡임 없음).

**수정**

- **트리거 모드** 감시 중 첫 번째 템플릿 매칭이 N회 반복 실행과 달리 계속 실패하던 문제를 고쳤습니다. 감시할 때도 대상 창을 활성화하고, 캡처 전 ROI 안내 오버레이를 끄며, 감시 주기마다 세션 검색 영역을 초기화합니다.

- 기능 **단축키**가 동작하지 않던 경우를 고쳤습니다. 트리거 대기·루프 간격 대기처럼 워크플로가 잠깐 멈춘 뒤에도 단축키로 다시 시작·중지할 수 있습니다.
- **메모장**·**계산기**·**CPU 감시** 창을 열어 둔 상태에서도 게임 창에서 기능 단축키를 쓸 수 있습니다(블록 편집 등 편집 창이 열려 있을 때는 기존처럼 막힙니다).
- PIPBONG 창이 활성화된 상태에서 단축키를 누르면, 제목 표시줄에 **대상 프로그램 창을 활성화한 뒤** 누르라는 안내가 잠깐 표시됩니다.

- **트리거 모드**에서 템플릿이 잠깐 안 보여도 감시가 바로 끊기던 문제를 고쳤습니다. N회 반복처럼 첫 템플릿 매칭이 될 때까지 계속 감시합니다.

**변경**

- 드래그 중 목록 **가장자리 자동 스크롤**의 **최소 속도**를 더 낮췄습니다. 목록 밖으로 살짝만 나갔을 때 거의 천천히 움직입니다.

- 드래그 중 목록 **가장자리 자동 스크롤**·**마우스 휠** 민감도를 이전보다 절반으로 더 낮췄습니다.

## v0.8.160–0.8.169 (2026/07/16)

**변경**

- 드래그 중 목록 **가장자리 자동 스크롤** 속도를 낮췄습니다. 목록 밖으로 살짝만 나가도 너무 빠르게 움직이지 않고, 멀리 나갈수록 천천히 빨라집니다.

- 드래그 중 목록 **밖**으로 커서를 옮겨도 자동 스크롤이 멈추지 않도록 고쳤습니다. 드래그를 잡은 채 **마우스 휠**로도 다시 스크롤됩니다(기능·워크플로·프로필 목록 등).

**추가**

- 드래그 중 목록 스크롤이 프로그램 전반의 기본 동작입니다. **기능 목록**·**라이브러리**·**프로필 목록**, **워크플로 블록** 순서 변경·**구간 반복** 구간 선택, **계산기** 표 셀 이동·수식 셀 고르기에서 모두 **위·아래 가장자리 자동 스크롤**과 **드래그 중 마우스 휠** 스크롤을 사용합니다.
- 워크플로 블록을 드래그해 들고 있는 동안 **마우스 휠**로 목록을 스크롤할 수 있습니다. 삽입 위치 안내선도 함께 갱신됩니다.
- 워크플로 블록 목록을 드래그해 순서를 바꿀 때, 목록 **위·아래 가장자리**에 커서를 대면 자동으로 스크롤되어 긴 목록도 끌어서 옮길 수 있습니다.

**수정**

- 워크플로 **블록 목록**에서 블록을 잡고 옮기는 동안 **마우스 휠**로 목록을 스크롤할 수 있습니다(Windows에서 드래그 중 휠이 먹지 않던 문제 수정).

- **라이브러리**를 접은 상태에서는 기능 목록과 라이브러리 사이 높이 조절(스플리터)을 할 수 없습니다.

- 드래그 중 목록 **밖**으로 커서를 옮겨도 자동 스크롤이 멈추지 않도록 고쳤습니다. 드래그를 잡은 채 **마우스 휠**로도 다시 스크롤됩니다(기능·워크플로·프로필 목록 등).

- **서브 창으로 지정**한 창에서도 템플릿 캡처·ROI 미리보기·매칭 테스트가 동작합니다. 주 게임 창이 꺼져 있거나 서브 창(런처 등)만 실행 중일 때 자동으로 서브 창을 찾습니다.

## v0.8.150–0.8.159 (2026/07/16)

**변경**

- 하단 **메모장** / **CPU 감시** / **계산기** 버튼을 다시 누르면 이미 열려 있는 창이 닫힙니다(토글).

**수정**

- **메모장**을 닫아도 다음 실행 때 자동으로 다시 열리던 문제를 고쳤습니다.

**변경**

- **메모장** 색상을 앱 테마(밝기/다크 모드)에 맞게 조정했습니다. 노란 포스트잇 색 대신 다른 패널과 같은 톤의 카드형 창입니다.

- **메모장**을 테두리 없는 카드형 창으로 정리했습니다. 상단을 드래그해 옮길 수 있고, 텍스트 영역은 깔끔하게 한 덩어리로 보입니다.

- **메모장** 열림/닫힘 상태가 프로필마다 따로 저장됩니다. 프로필을 바꾸면 해당 프로필에 저장된 상태(켜짐/꺼짐)에 맞게 메모 창이 표시되거나 숨겨집니다.

**추가**

- **메모장**을 켜 둔 상태로 종료하면 다음 실행 때도 메모 창이 자동으로 열립니다. 메모 창을 닫으면 닫힌 상태가 저장됩니다.

- 기능 목록과 **라이브러리** 사이에 세로 스플리터를 두어, 라이브러리를 펼친 상태에서 드래그로 각 영역 높이를 조절할 수 있습니다. 크기는 재시작 후에도 유지됩니다.

**변경**

- 메인 창 **로그 / 대상 창** 사이 가로 스플리터를 더 아래로 내려 워크플로우 영역을 넓힐 수 있습니다(하단 패널 최소 높이 완화).
- **창 목록**에서 현재 **주 대상**·**서브 창**으로 지정된 항목이 색 배경·굵은 글씨·**● 주 대상** / **● 서브 창** 배지로 표시됩니다.

**추가**

- **창 목록**에서 창을 고른 뒤 **서브 창으로 지정**으로 프로필 서브 대상 창을 설정할 수 있습니다. 서브 창이 포커스된 상태면 대상 창 정보에 **● 서브 창**으로 표시됩니다.

**변경**

**변경**
