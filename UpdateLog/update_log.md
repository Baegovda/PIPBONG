# PIPBONG 업데이트 로그

버전별 변경 사항을 한글로 정리한 문서입니다. 최신 릴리즈는 [GitHub Releases](https://github.com/Baegovda/PIPBONG/releases/latest)에서 받을 수 있습니다.

개발용 상세 기록(영문)은 [`AGENTS.md`](../AGENTS.md) §11을 참고하세요.  
패치 버전은 **10개 단위**로 묶어 표기합니다(예: `v0.8.60–0.8.69`).

---

## v0.8.70–0.8.79 (2026/07/10)

**변경**

- 업데이트 로그를 패치 10개 단위로 묶어 표기합니다(예: `v0.8.60–0.8.69`). 같은 구간에 새 버전이 나오면 해당 섹션에 항목을 추가합니다.
- **F5** 실행을 일반적인 C++ 개발 방식으로 되돌렸습니다 — 빌드 후 `build/Release/PIPBONG.exe` 실행(디버거 연결). **Ctrl+Shift+B**는 빌드만 수행합니다.

**수정**

- **루프 간격**이 안 먹는 것처럼 보이던 문제: **랜덤**인데 최소·최대가 비어 있으면(0~0) 실제로는 0ms로 동작하던 경우를 고쳤고, 숫자 입력 후 바로 확인해도 값이 반영되도록 했습니다. 실행 로그에 **루프 간격 Nms 대기**가 표시됩니다.
- **누를 동안** 단축키와 워크플로가 같은 키(예: **Q** 홀드 + **Q 탭**)일 때, 합성 키 뗌이 홀드를 끊어서 루프가 매번 1회부터 다시 시작하고 루프 간격이 무시되던 문제를 수정했습니다.
- **누를 동안** 단축키와 워크플로 키보드가 같으면, 게임은 손가락으로 누르는 동안 키가 계속 눌린 것으로 보므로 루프 간격이 체감되지 않습니다. 저장·실행 시 경고를 표시합니다.
- 키보드 블록 **탭**(예: **Q** 탭)이 게임에서 가끔 인식되지 않던 문제를 개선했습니다 — 하드웨어 스캔코드로 보내고, 누름·뗌 사이에 짧은 유지 시간을 둡니다.
- **누를 동안** 단축키와 워크플로 키보드가 같아도(예: **Q** 홀드 + **Q** 탭) 동작합니다 — 이미 눌린 키는 잠깐 뗐다가 다시 눌러 게임에서 탭·루프 간격이 체감되도록 했고, 불가능하다고 안내하던 경고는 제거했습니다.
